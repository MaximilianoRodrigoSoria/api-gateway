API Gateway

Único punto de entrada por delante de un conjunto de microservicios independientes: resuelve rate limiting, autenticación y enrutamiento una sola vez. Cada servicio es autónomo y dueño de su base de datos.

Java 21SpringResilience4j RedisFlywayDocker TestcontainersPrometheus + GrafanaHexagonal
Ver en GitHub Documentación

Qué resuelve

Single entry point. Los servicios internos no se exponen; todo entra por /gateway/**.
Rate limiting distribuido por cliente / API key con Resilience4j y Redis.
Auth centralizada. Verificación de autenticación una sola vez en el borde.
Routing por prefijo más largo hacia cada microservicio (RestClient).
Resiliencia: timeouts, retry y circuit breaker por ruta.
Trazabilidad end-to-end propagando X-Correlation-Id.
